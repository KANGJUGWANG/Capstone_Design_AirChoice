# v0.4 Product — 앱 구조 및 화면 기준

---

## 1. 문서 목적

본 문서는 v0.4 기준 프로젝트의 결과물 형태를 정리한다.

본 문서의 목적은 아래와 같다.

- 프로젝트 결과물을 분석 대시보드형 앱으로 정의
- 핵심 화면 구성과 기능 범위 정리
- 데이터 파이프라인과 화면 요소 간 연결 기준 정리
- 현재 프로토타입의 역할과 제한 사항 명시

---

## 2. 결과물 정의

v0.4 기준 결과물은 단순 모델 실험 결과가 아니다.

최종 결과물은 아래 두 축을 함께 포함한다.

| 축 | 내용 |
|---|---|
| 데이터/분석 축 | 항공권 가격 수집, BUY/WAIT 판단, 가격 흐름 분석 |
| 서비스/화면 축 | 사용자가 현재 가격 상태와 기다림 가치를 직관적으로 확인할 수 있는 대시보드형 앱 |

즉 본 프로젝트는  
**항공권 가격 데이터를 수집·분석하고, 그 결과를 사용자에게 BUY/WAIT 의사결정 보조 형태로 보여주는 서비스형 캡스톤 결과물**로 해석한다.

---

## 3. 제품 목표

현재 기준 제품 목표는 아래와 같다.

- 사용자가 특정 노선과 날짜 조건에서 현재 가격 상태를 확인할 수 있어야 함
- 현재 시점에서 BUY / WAIT 판단을 직관적으로 확인할 수 있어야 함
- 판단 이유를 요인 분석 형태로 설명할 수 있어야 함
- 가격 변화 감지 및 알림 설정이 가능해야 함
- 결과 화면이 단순 수치 나열이 아니라 행동 보조형 UI로 구성되어야 함

---

## 4. 핵심 사용자 질문

앱은 아래 질문에 답하는 방향으로 구성한다.

- 지금 가격은 높은 편인가, 낮은 편인가
- 지금 사는 것이 더 유리한가
- 더 기다릴 가치가 남아 있는가
- 이 판단에 영향을 주는 핵심 요인은 무엇인가
- 가격이 바뀌면 다시 알림을 받을 수 있는가

---

## 5. 핵심 화면 구성

현재 기준 핵심 화면은 아래 5개 축으로 정리한다.

| 화면 | 역할 |
|---|---|
| 검색 & 요약 | 노선/날짜 입력, 현재 가격 상태 요약 |
| 가격 분석 / BUY-WAIT | 현재 가격과 BUY/WAIT 판단 표시 |
| 요인 분석 | 현재 판단의 주요 요인 설명 |
| 알림 설정 | 목표가, 골든타임, 관심 노선 알림 설정 |
| 환경 설정 | 수집/표시 기준 및 사용자 환경 설정 |

---

## 6. 화면별 기준 기능

## 6-1. 검색 & 요약 화면

역할:
- 사용자가 노선과 날짜 조건을 입력
- 현재 관측 가격 상태를 한눈에 요약

핵심 기능:
- 출발지 / 도착지 입력
- 출발일 / 귀국일 입력
- 편도 / 왕복 선택
- 현재 가격 요약 카드
- 급락 노선 또는 추천 노선 카드
- 최근 수집 기준 요약 정보 표시

핵심 출력:
- 현재 가격
- 최근 변동 방향
- DPD
- 직항 여부
- 항공사 태그 요약

---

## 6-2. 가격 분석 / BUY-WAIT 화면

역할:
- 현재 시점의 핵심 판단을 가장 직접적으로 보여주는 화면

핵심 기능:
- BUY / WAIT 결과 표시
- 점수 또는 확률 형태의 판단 보조값 표시
- 최근 관측 흐름 시각화
- 현재 가격의 상대적 위치 표시

핵심 출력:
- BUY / WAIT 라벨
- 판단 점수
- 현재 가격
- 최근 가격 변화
- 위험/기회 배지

현재 기준 해석:
- 이 화면은 “최적 구매일 추천”이 아니라 **현재 시점의 행동 판단 보조**를 제공한다.

---

## 6-3. 요인 분석 화면

역할:
- 현재 BUY/WAIT 판단의 근거를 설명

핵심 기능:
- 주요 feature 요약
- 가격 변동에 영향을 준 요인 설명
- 직항/경유, 항공사, DPD, 요일 등 조건 표시
- 현재 판단에 유리/불리하게 작용한 요인 구분

핵심 출력:
- DPD 요인
- 항공사 태그
- 출발 요일
- 직항 여부
- 최근 가격 추세
- 설명 문장 또는 요약 카드

현재 기준 해석:
- 이 화면은 예측값 제시보다 **설명 가능성 확보**에 목적을 둔다.

---

## 6-4. 알림 설정 화면

역할:
- 사용자가 반복 확인하지 않아도 가격 변화를 추적할 수 있도록 지원

핵심 기능:
- 목표 가격 알림
- BUY/WAIT 상태 변경 알림
- 급락 감지 알림
- 노선별 관심 설정
- 골든타임 알림 설정

핵심 출력:
- 활성 알림 목록
- 알림 조건
- 최근 트리거 이력

현재 기준 해석:
- 알림 기능은 모델 고도화 이후 확장 가능하나, v0.4에서는 구조만 우선 정리한다.

---

## 6-5. 환경 설정 화면

역할:
- 시스템 및 사용자 기준 설정

핵심 기능:
- 수집 기준 설명
- 가격 표시 단위
- 직항 우선 여부
- 알림 정책 설정
- 기본 노선/기본 여행 유형 저장

핵심 출력:
- 사용자 설정값
- 시스템 운영 기준 요약
- 알림/표시 관련 설정

---

## 7. 데이터와 화면 연결 기준

화면은 raw 데이터를 직접 읽지 않는다.  
화면 표시 기준은 정규화된 processed 계층 또는 DB 조회 결과로 둔다.

| 화면 | 주 데이터 소스 |
|---|---|
| 검색 & 요약 | `flight_observations` 요약 조회 |
| 가격 분석 / BUY-WAIT | observation + 판단 결과 |
| 요인 분석 | feature 요약 + 설명 결과 |
| 알림 설정 | alert 조건 저장 구조 |
| 환경 설정 | 사용자/시스템 설정값 |

현재 기준 원칙:
- 화면은 raw 인터셉트 데이터에 직접 의존하지 않음
- 앱은 정규화 후 저장된 데이터 계층을 소비함
- UI는 데이터 파이프라인의 마지막 출력 계층으로 해석함

---

## 8. 현재 프로토타입 상태

현재 프로토타입은 Manus 기반으로 생성되었으며,  
화면 구성 방향과 주요 기능 축을 확인하기 위한 참고 자산으로 사용한다.

현재 상태 해석은 아래와 같다.

| 항목 | 상태 |
|---|---|
| 화면 초안 | 존재 |
| 상호작용 흐름 | 부분 구현 |
| 최적화 수준 | 낮음 |
| 최종 앱 여부 | 아님 |
| 역할 | 구조/방향 확인용 프로토타입 |

현재 프로토타입은 **기능 방향과 결과물 형태를 설명하는 참고물**이며,  
최종 산출물로 직접 간주하지 않는다.
