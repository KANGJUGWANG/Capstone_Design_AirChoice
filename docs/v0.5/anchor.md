# v0.5 Project Anchor

---

## 1. 문서 목적

본 문서는 v0.4.3 이후 앱 개발이 본격 시작되고 Vercel 배포까지 완료된 시점을 기준으로 작성한다.

v0.5는 수집 인프라 구축(v0.4.x) 이후 앱 개발·배포 단계 진입을 반영하는 버전이다.

---

## 2. 상태 구분 기준

- **확정**: 현재 운영 기준으로 채택한 항목
- **보류**: 후보로 유지하되 현재 메인 진행 대상이 아닌 항목
- **미확정**: 후속 결정 또는 구현 결과 확인 후 결정할 항목
- **폐기**: 이전 버전에서 검토했으나 현재 진행 대상에서 제외한 항목

---

## 3. v0.4.3 대비 주요 변경 사항

| 구분 | v0.4.3 상태 | v0.5 상태 |
|---|---|---|
| 프론트엔드 | 구조 설계 완료, 구현 미착수 | 8개 화면 전부 구현 완료 |
| 백엔드 | 설계 단계 | FastAPI 구조 완성, 카카오 로그인·저장 기능 구현 |
| 배포 | 미배포 | Vercel deploy 브랜치 Production 고정 완료 |
| 카카오 로그인 | 앱 등록 진행 중 | 동작 확인 완료 |
| 데이터 저장 방식 | MySQL 연결 예정 | 현재 JSON 파일 기반 store 운영 (MySQL 연결 미완료) |
| 수집 운영 | 운영 중 (2026-04-16~) | 운영 지속 중 (~11일치 누적) |

|---|---|---|
| 기술 스택 | React + Vite | 확정 |

| 구현 화면 | HomePage, SearchResultPage, CardDetailPage, SearchDetailPage, SavedListPage, SettingsPage, ModelInfoPage, LoginPage, AuthCallbackPage (총 8종) | 확정 |

## 5. 현재 미확정 항목

- 학교 서버 스펙 (백엔드 서버 위치 결정 블로커)
- FastAPI ↔ MySQL 연결 전환 (현재 JSON 파일 기반)
- 프론트 ↔ FastAPI 실제 API 연동 (현재 Mock 데이터 또는 로컬 API)
- 앱 이름 / 로고 / 포인트 컬러
- 비로그인 사용자 허용 여부
- BUY/WAIT 라벨 기준 및 모델 구조
- 카드 상세보기 세부 구성 (모델 완성 후 확정)
- Kaggle 보조 데이터 최종 폐기 여부

## 6. 현재 폐기 항목
