# v0.4.1 Project Anchor

---

## 1. 문서 목적

본 문서는 v0.4 이후 수집 코드 구현, DB 스키마 서버 적용, 서버 인프라 구성이 완료된 시점을 반영하여 현재 기준점(Anchor)으로 사용한다.

v0.4.1은 신규 방향 전환이 아니라, v0.4에서 설계한 구조가 실제 구현 단계를 통과한 상태를 반영하는 버전이다.

다음 목적을 가진다.

- v0.4 대비 확정으로 전환된 항목 명시
- 서버 인프라 및 DB 구성 완료 상태 기록
- 수집 코드 구현 완료 상태 기록
- 서버 운영 시작 전 상태를 명확하게 구분
- 이후 v0.4.2 (운영 시작 단계) 기준 문서와 구분점 설정

---

## 2. 상태 구분 기준

- **확정**: 현재 운영 기준으로 채택한 항목
- **보류**: 후보로 유지하되 현재 메인 진행 대상이 아닌 항목
- **미확정**: 후속 검증 또는 구현 결과 확인 후 결정할 항목
- **폐기**: 이전 버전에서 검토했으나 현재 진행 대상에서 제외한 항목

## 3. v0.4 대비 주요 변경 사항

| 구분 | v0.4 상태 | v0.4.1 상태 |
|---|---|---|
| 수집 코드 | 설계 방향 확정, 구현 중 | 편도 + 왕복 모두 구현 완료 (로컬 테스팅 완료) |
| DB 스키마 | 설계 중, 서버 미적용 | 서버 적용 완료 (ALTER TABLE 포함) |
| 서버 인프라 | 운영 계획 수준 | Docker 기동, MySQL 정상 확인, 디렉터리 구조 완료 |
| 왕복 수집 구조 | 별도 테이블 검토 중 | 단일 테이블 ret_ 컬럼 확장 방식으로 확정 |
| 수집 실행 | 미시작 | 로컬 테스팅 완료, 서버 운영 대기 상태 |
| cron 등록 | 계획 수준 | 서버 테스팅 후 등록 예정 |
| Kaggle 보조 데이터 | 조건부 검토 유지 | 불필요 가능성 높음으로 판단 전환 |
| 와이어프레임 | 프로토타입 초안 | 5종 확정, MVP 범위는 다음 단계 확정 예정 |
| schema.md | 설계 초안 | 실제 서버 구조 기준으로 전면 갱신 |

| 항목 | 내용 | 상태 |
|---|---|---|
| 수집 방식 | Google Flights GetBookingResults 인터셉트 기반 | 확정 |

| 수집 주기 | 1일 3회 (8시간 간격) | 확정 |
